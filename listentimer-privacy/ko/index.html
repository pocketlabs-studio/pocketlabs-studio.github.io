개인정보처리방침

시행일: 2026년 3월 22일 | 최종 수정일: 2026년 4월 23일

PocketLabs(이하 "회사")는 ListenTimer(이하 "앱") 이용자의 개인정보를 중요시하며, 「개인정보 보호법」 및 관련 법령을 준수합니다. 본 개인정보처리방침은 앱이 수집하는 개인정보의 항목, 수집 목적, 보유 기간, 제3자 제공 등을 안내합니다.

1. 수집하는 개인정보 항목

구분 수집 항목 저장 위치
앱 설정 타이머 프리셋, 음성 캐릭터, 음성 간격, 햅틱 설정 기기 내 로컬 저장 (서버 전송 없음)
광고 데이터 (무료 사용자) 광고 식별자(IDFA), 디바이스 정보, 앱 사용 데이터 Google LLC 서버
앱 사용 통계 (Firebase Analytics) 앱 인스턴스 ID(가명 식별자), 디바이스 정보(기기 모델, OS 버전, 언어/지역), 자동 수집 이벤트(첫 실행, 세션 시작·종료, 앱 업데이트, 앱 제거, 인앱 결제 등), IP 기반 대략적 위치(국가/지역 수준) Google LLC (Firebase) 서버
크래시 리포트 (Firebase Crashlytics) 크래시 스택 트레이스, 디바이스 상태(기기 모델, OS 버전, 메모리·디스크 사용량, 배터리 상태 등), Crashlytics 설치 UUID(가명 식별자), 크래시 발생 시각, 앱 실행 시간, 비치명적 에러 로그 Google LLC (Firebase) 서버
본 앱은 자체 서버를 운영하지 않으며, 회원가입, 로그인, 계정 정보를 수집하지 않습니다. Firebase Analytics 및 Crashlytics가 사용하는 식별자(앱 인스턴스 ID, Crashlytics 설치 UUID)는 가명 식별자로, 사용자의 이름·이메일·전화번호 등 개인을 직접 식별할 수 있는 정보와 연결되지 않습니다.

2. 개인정보 수집 방법

  • 앱 설정: 사용자가 앱 내에서 직접 설정한 값이 기기에 자동 저장됩니다.
  • 광고 데이터: Google AdMob SDK가 자동 수집하며, iOS의 앱 추적 투명성(ATT) 동의에 기반합니다. ATT에서 추적을 허용하지 않으면 IDFA를 수집하지 않습니다.
  • 앱 사용 통계 (Firebase Analytics): Firebase Analytics SDK가 앱 실행·사용 과정에서 자동으로 수집합니다. IDFA를 사용하지 않으며, Firebase가 자체 발급한 가명 식별자(앱 인스턴스 ID)에 기반합니다.
  • 크래시 리포트 (Firebase Crashlytics): 앱이 비정상 종료(크래시)되거나 비치명적 에러가 발생한 경우, Firebase Crashlytics SDK가 다음 앱 실행 시 자동으로 리포트를 업로드합니다.

3. 개인정보 수집 및 이용 목적

  • 앱 설정: 사용자 맞춤 타이머 환경 제공
  • 광고 데이터: 맞춤형 광고 제공 및 광고 성과 측정
  • 앱 사용 통계 (Firebase Analytics): 앱 사용 패턴 분석을 통한 기능 개선, 성능 최적화, 사용자 경험 향상
  • 크래시 리포트 (Firebase Crashlytics): 앱 안정성 진단 및 버그 수정, 품질 개선

4. 개인정보의 보유 및 이용 기간

구분 보유 기간
앱 설정 (로컬) 앱 삭제 시 자동 삭제
광고 데이터 Google 개인정보처리방침에 따름
Firebase Analytics 데이터 Google Analytics for Firebase 기본 보존 기간(최대 14개월) 적용. 자세한 내용은 Firebase 데이터 보존 정책 참조
Firebase Crashlytics 데이터 크래시 발생일로부터 최대 90일간 보존 후 자동 삭제 (Firebase 기본 정책)

5. 개인정보의 제3자 제공

제공 받는 자 제공 항목 제공 목적
Google LLC (AdMob) 광고 식별자(IDFA), 디바이스 정보, 앱 사용 데이터 맞춤형 광고 제공 및 광고 성과 측정
Google LLC (Firebase Analytics) 앱 인스턴스 ID, 디바이스 정보, 자동 수집 이벤트, IP 기반 대략적 위치 앱 사용 통계 분석 및 기능 개선
Google LLC (Firebase Crashlytics) 크래시 스택 트레이스, 디바이스 상태, Crashlytics 설치 UUID 크래시 진단 및 앱 안정성 개선

프리미엄 구매 시 광고가 제거되어 AdMob에 대한 데이터 제공이 중단됩니다. 단, Firebase Analytics 및 Crashlytics는 앱 품질 개선 목적으로 프리미엄 사용자에게도 동일하게 동작합니다.

6. 개인정보의 국외 이전

광고 데이터, Firebase Analytics 데이터, Firebase Crashlytics 데이터는 Google LLC의 서버(미국 등)로 전송됩니다. 자세한 내용은 Google 개인정보처리방침 및 Firebase 개인정보 및 보안을 참조하세요.

8. 개인정보의 파기 절차 및 방법

  • 앱 설정 (로컬 데이터): 앱을 삭제하면 기기에 저장된 모든 설정 데이터가 자동으로 삭제됩니다.
  • 광고 데이터: Google의 데이터 보유 정책에 따라 처리됩니다. 사용자는 ATT 설정 변경을 통해 향후 데이터 수집을 중단할 수 있습니다.
  • Firebase Analytics 데이터: Firebase 기본 보존 기간(최대 14개월) 경과 시 자동 삭제됩니다. 앱 삭제 시 향후 데이터 수집은 중단되며, 기존에 수집된 데이터의 즉시 삭제는 아래 문의 이메일을 통해 요청할 수 있습니다.
  • Firebase Crashlytics 데이터: 크래시 발생일로부터 90일 경과 시 자동 삭제됩니다. 즉시 삭제 요청은 아래 문의 이메일로 가능합니다.

10. 안전성 확보 조치

회사는 개인정보의 안전성 확보를 위해 다음과 같은 조치를 취하고 있습니다.

  • 자체 서버를 운영하지 않아 서버 해킹·유출 위험이 없습니다.
  • 모든 앱 설정은 기기 내 로컬에만 저장됩니다.
  • 광고 네트워크 및 Firebase와의 통신은 HTTPS 암호화를 통해 전송됩니다.
  • Firebase Analytics 및 Crashlytics는 가명 식별자만 사용하며, 사용자의 이름·이메일·전화번호 등 직접 식별 정보를 수집하지 않습니다.

11. 14세 미만 아동의 개인정보

본 앱은 14세 미만 아동의 개인정보를 의도적으로 수집하지 않습니다. 만약 14세 미만 아동의 개인정보가 수집된 사실을 알게 된 경우, 해당 정보를 지체 없이 삭제하겠습니다.

12. 프리미엄 구매 시 변경 사항

프리미엄을 구매하면 앱 내 광고가 제거됩니다. 이에 따라 AdMob SDK를 통한 광고 데이터 수집(IDFA, 디바이스 정보, 앱 사용 데이터)이 중단됩니다.

단, Firebase Analytics(앱 사용 통계) 및 Firebase Crashlytics(크래시 리포트)는 앱의 안정성과 품질 개선 목적으로 프리미엄 사용자에게도 동일하게 동작합니다.

13. 동의 철회 방법

광고 추적에 대한 동의는 아래 경로에서 언제든지 변경할 수 있습니다.

iOS: 설정 > 개인정보 보호 및 보안 > 추적 > ListenTimer 토글 끄기

추적을 비활성화하면 IDFA 기반의 맞춤형 광고 데이터 수집이 중단됩니다.

Firebase Analytics 및 Crashlytics 데이터 수집은 ATT 설정과는 별개로 동작하며, 현재 앱 내 별도 비활성화 옵션은 제공되지 않습니다. 데이터 수집을 완전히 중단하고자 하는 경우 앱을 삭제할 수 있으며, 기존에 수집된 데이터의 즉시 삭제는 아래 9. 개인정보 보호책임자 항목의 이메일을 통해 요청할 수 있습니다.

14. 개인정보처리방침 변경

본 개인정보처리방침이 변경되는 경우, 변경 사항을 이 페이지에 게시하고 시행일을 업데이트합니다. 중요한 변경 사항이 있을 경우 앱 내 공지를 통해 안내할 수 있습니다.
